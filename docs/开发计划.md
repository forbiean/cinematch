# CineMatch 项目开发计划（渐进式落地）

基于当前项目状态（前后端开发环境已完成，前端仍以 mock 数据为主），按“先简单、每次改动小、逐步替换假数据”的原则推进。

## 开发原则

1. 每个迭代只完成一个小功能闭环。
2. 每次都包含：后端接口 + 前端接入 + 最小验证。
3. 先替换读接口（风险低），再做写接口（评分/收藏），最后做推荐和后台。
4. 每完成一步就移除对应页面 mock 依赖，避免双数据源长期并存。

## 阶段 0：项目基线与规范（最小启动）

1. 统一 API 响应结构与错误码（如 `code/message/data`）。
2. 增加后端全局异常处理、参数校验返回格式。
3. 前端封装统一 `api client`（`/api`、超时、错误提示）。
4. 验收：前端调用 `/api/health` 成功并展示“服务可用”。

## 阶段 1：电影只读链路（最简单、最先替换 mock）

### 功能 1：电影列表接口（分页/搜索/分类/排序）
- 后端：`GET /api/movies`
- 前端：替换 `MoviesPage` mock 列表与分页。
- 验收：列表页所有筛选与分页都来自后端。

### 功能 2：电影详情接口
- 后端：`GET /api/movies/{id}`
- 前端：替换 `MovieDetailPage` 的电影详情数据源。
- 验收：详情页刷新后仍能正确展示，不依赖本地 mock。

### 功能 3：首页“本周热门”
- 后端：`GET /api/movies?sort=hot`（或单独 `/api/movies/hot`）
- 前端：替换 `HomePage` 热门区 mock。
- 验收：首页热门电影来自数据库。

## 阶段 2：账号体系（先登录注册，不上复杂权限）

### 功能 4：用户注册
- 后端：`POST /api/auth/register`
- 前端：`RegisterPage` 表单提交到后端。
- 验收：注册成功写入 `users`。

### 功能 5：用户登录（JWT）
- 后端：`POST /api/auth/login` 返回 token。
- 前端：保存 token（localStorage）并在请求头携带。
- 验收：登录后可访问需要登录的接口。

### 功能 6：当前用户信息
- 后端：`GET /api/me/profile`
- 前端：`ProfilePage` 顶部用户信息改为真实数据。
- 验收：不同账号看到各自信息。

## 阶段 3：用户行为（评分/收藏，逐步替换详情与个人页假数据）

### 功能 7：评分提交与覆盖更新
- 后端：`POST /api/movies/{id}/ratings`（同一用户同电影只保留一条）。
- 前端：详情页评分星星提交后端。
- 验收：刷新后评分状态正确回显。

### 功能 8：收藏/取消收藏
- 后端：`POST /api/movies/{id}/favorite`、`DELETE /api/movies/{id}/favorite`。
- 前端：详情页收藏按钮真实生效。
- 验收：收藏状态刷新可持久化。

### 功能 9：个人中心“我的评分”“我的收藏”
- 后端：`GET /api/me/ratings`、`GET /api/me/favorites`。
- 前端：替换 `ProfilePage` 对应 mock 区块。
- 验收：个人中心行为数据全部来自后端。

## 阶段 4：推荐（先规则版，满足 MVP）

### 功能 10：推荐接口 v1（标签偏好 + 冷启动热门）
- 后端：`GET /api/recommendations`，返回 `score/reason`。
- 前端：`RecommendationsPage` 全量接入后端。
- 验收：推荐页无 mock，能展示推荐理由。

### 功能 11：推荐过滤规则
- 后端：过滤已评分/已收藏，尽量返回 10 条（不足则返回可用最大值）。
- 前端：空态展示（可推荐不足时提示）。
- 验收：推荐结果符合 PRD 业务规则。

## 阶段 5：后台最小可用（内容维护优先）

### 功能 12：后台电影列表
- 后端：`GET /api/admin/movies`
- 前端：`AdminPage` 电影表格改真实数据。
- 验收：后台可查看电影数据。

### 功能 13：新增电影
- 后端：`POST /api/admin/movies`
- 前端：接入“新增电影”按钮（先简版表单）。
- 验收：新增后列表可见。

### 功能 14：编辑电影
- 后端：`PATCH /api/admin/movies/{id}`
- 前端：接入“编辑”按钮。
- 验收：修改后详情页与列表页同步变化。

## 阶段 6：质量补强（每步小修，不阻塞主线）

### 功能 15：基础权限分离
- 普通用户不可调用 admin 接口。

### 功能 16：关键接口参数校验与错误提示统一
- 全局异常与前端提示一致化。

### 功能 17：最小测试集
- 后端：movies/auth/ratings/favorites/recommendations 核心用例。
- 前端：关键页面联调冒烟测试清单。

## 建议迭代节奏

1. 迭代 1：功能 1（电影列表）
2. 迭代 2：功能 2（电影详情）
3. 迭代 3：功能 4（注册）
4. 迭代 4：功能 5（登录）
5. 迭代 5：功能 7（评分）
6. 迭代 6：功能 8（收藏）
7. 迭代 7：功能 9（个人中心行为数据）
8. 迭代 8：功能 10（推荐）
9. 迭代 9：功能 12（后台电影列表）
10. 迭代 10：功能 13/14（新增+编辑）
