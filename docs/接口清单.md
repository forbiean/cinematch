# 接口清单（CineMatch）

本文档汇总当前后端 API（基于 `backend/src/main/java/com/cinematch/api`）。

## 通用说明
- Base URL：`/api`
- 认证方式：`Authorization: Bearer <token>`
- 管理员接口：需要 `role=ADMIN`
- 统一返回：`application/json`

## 1. 健康检查

### GET `/api/health`
- 说明：服务存活检查
- 鉴权：否

## 2. 认证

### POST `/api/auth/register`
- 说明：用户注册（默认角色 USER）
- 鉴权：否
- 请求体：
```json
{
  "nickname": "张三",
  "email": "user@example.com",
  "password": "password123"
}
```

### POST `/api/auth/login`
- 说明：用户登录
- 鉴权：否
- 请求体：
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```
- 返回（示例字段）：`token`、`tokenType`、`expiresIn`、`userId`、`email`、`role`

## 3. 电影（前台）

### GET `/api/movies`
- 说明：电影列表
- 鉴权：否
- Query：
  - `page`（默认 1）
  - `pageSize`（默认 8）
  - `query`（可选，标题/简介关键词）
  - `genre`（默认 `all`）
  - `sort`（默认 `rating`，可选：`rating`/`year`/`title`/`hot`）

### GET `/api/movies/{id}`
- 说明：电影详情（含相似推荐、标签、导演、主演等）
- 鉴权：否

### POST `/api/movies/{id}/ratings`
- 说明：评分（1~5）
- 鉴权：是
- 请求体：
```json
{ "score": 5 }
```

### POST `/api/movies/{id}/favorite`
- 说明：收藏电影
- 鉴权：是

### DELETE `/api/movies/{id}/favorite`
- 说明：取消收藏
- 鉴权：是

## 4. 推荐

### GET `/api/recommendations`
- 说明：获取个性化推荐
- 鉴权：是
- 备注：会写入 `recommendation_logs`（推荐日志）

## 5. 个人中心

### GET `/api/me/profile`
- 说明：个人资料概览（昵称、邮箱、加入时间、统计）
- 鉴权：是

### PATCH `/api/me/profile`
- 说明：修改个人资料
- 鉴权：是
- 请求体：
```json
{
  "nickname": "新昵称",
  "password": "newPassword123"
}
```
- 说明：
  - `nickname` 可改
  - `password` 可选；如传入需至少 8 位

### GET `/api/me/ratings`
- 说明：我的评分列表
- 鉴权：是

### GET `/api/me/favorites`
- 说明：我的收藏列表
- 鉴权：是

## 6. 后台管理（管理员）

Base：`/api/admin`

### GET `/api/admin/dashboard`
- 说明：后台概览数据
- 鉴权：管理员
- 内容：
  - 电影/用户/评分/收藏率概览
  - 近 7 日评分趋势
  - 热门标签
  - 推荐效果概览
  - 最近推荐日志

### GET `/api/admin/movies`
- 说明：后台电影列表
- 鉴权：管理员
- Query：
  - `page`（默认 1）
  - `pageSize`（默认 10）
  - `query`（可选，标题/简介关键词）

### POST `/api/admin/movies`
- 说明：新增电影
- 鉴权：管理员
- 请求体：
```json
{
  "title": "电影名",
  "year": 2024,
  "poster": "https://...",
  "director": "导演名",
  "cast": ["主演A", "主演B"],
  "summary": "简介",
  "tags": ["科幻", "剧情"]
}
```

### PATCH `/api/admin/movies/{id}`
- 说明：编辑电影
- 鉴权：管理员
- 请求体字段同新增电影（均可按需传递）

### DELETE `/api/admin/movies/{id}`
- 说明：删除电影（不可逆）
- 鉴权：管理员

### GET `/api/admin/tags`
- 说明：标签列表（含使用次数）
- 鉴权：管理员

### POST `/api/admin/tags`
- 说明：新增标签
- 鉴权：管理员
- 请求体：
```json
{ "name": "科幻" }
```

### DELETE `/api/admin/tags/{name}`
- 说明：删除标签（会从电影标签关系中移除该标签）
- 鉴权：管理员

## 7. 常见状态码
- `200`：成功
- `400`：参数错误
- `401`：未登录或 token 无效
- `403`：权限不足（非管理员访问后台）
- `404`：资源不存在
- `409`：资源冲突（如邮箱已注册）
- `500`：服务端异常
